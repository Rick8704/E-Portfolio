## 📖 What I Learned

Computer Organization & Architecture moved from logic gates to the internal structure of computers — registers, memory hierarchies, instruction cycles, and CPU design. This course connected the digital logic from the previous semester to how a real processor operates.

Human-Computer Interaction introduced principles of usability, interface design, and user-centred thinking. Probability & Statistical Data Analysis built the mathematical foundation for reasoning about data — a skill that becomes essential in data engineering.

Programming Technique II deepened C++ knowledge with object-oriented concepts, pointers, and dynamic memory. System Analysis & Design (WBL) applied structured analysis methods to real scenarios, covering requirements gathering, data flow diagrams, and system modelling.

---

## 💭 Reflection

This semester reinforced how interconnected the different areas of computing are. Understanding computer architecture made programming feel more meaningful — knowing what happens at the hardware level when code runs changed how I think about efficiency.

The WBL component of System Analysis & Design was a useful introduction to working with requirements and communicating technical designs clearly. Probability & Statistical Data Analysis was challenging but provided tools that I would later use extensively in data-focused courses.

By the end of Year 1, I had a clearer picture of where my interests lay — towards data, systems, and practical problem-solving rather than pure hardware or theory.
