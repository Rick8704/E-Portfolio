# 📘 Programming Technique II

**Year 1 · Semester 2**

This course extended C++ programming into object-oriented concepts — classes, inheritance, polymorphism, encapsulation, pointers, dynamic memory allocation, and file handling.

---

## 📂 Work

No assignment or lab subfolders recorded for this course.

---

## 📖 What I Learned

Programming Technique II deepened my C++ knowledge by introducing object-oriented programming. I learned to design classes with clear responsibilities, use inheritance to model relationships between types, apply polymorphism to write flexible code, and manage memory manually using pointers and dynamic allocation.

File handling introduced the ability to read and write persistent data — connecting programs to external information sources for the first time.

---

## 💭 Reflection

**What I gained:** A strong grounding in OOP that underpins most of the software development work I have done since. Understanding encapsulation and inheritance made it much easier to transition to Java, Python, and Dart in later courses.

**Suggestions for improvement:** More emphasis on common design patterns — such as factory, observer, or strategy — would have given practical frameworks for applying OOP principles in larger programs.

**Other thoughts:** The jump from structured programming (PT I) to object-oriented design (PT II) was significant. It required a shift in how I think about programs — from sequences of instructions to collections of interacting objects with defined responsibilities. That shift took time, but the OOP mindset has been central to almost all development work since.
