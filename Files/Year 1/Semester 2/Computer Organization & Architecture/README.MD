## 📖 What I Learned

Computer Organization & Architecture bridged the gap between digital logic and modern programming. I learned how instructions are fetched, decoded, and executed in a CPU, how memory is organised into caches and registers, and how assembly language relates to C code. The labs provided hands-on practice with arithmetic circuits and hardware-level concepts.

The group project required applying these principles to a real implementation, combining hardware understanding with practical programming.

---

## 💭 Reflection

**What I gained:** A mental model of what happens "below" the code — understanding the CPU, instruction cycle, and memory hierarchy made me more conscious of performance and efficiency in software development.

**Suggestions for improvement:** More opportunities to write and run assembly code would strengthen the connection between machine-level instructions and high-level programming. The theoretical content is easier to absorb when paired with practical execution.

**Other thoughts:** This course changed how I read code. Knowing roughly what the CPU does with each instruction made me more thoughtful about how I write loops, manage memory, and reason about program performance.
