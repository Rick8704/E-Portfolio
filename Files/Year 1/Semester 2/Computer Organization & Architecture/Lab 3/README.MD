## 📖 What I Did

I designed a circuit diagram using draw.io to document the hardware structure, then implemented corresponding C programs in Visual Studio. The lab required both a visual hardware representation and working C code, reinforcing the relationship between circuit design and software.

---

## 💭 Reflection

**What I gained:** Experience linking hardware design (circuit diagrams) with software implementation (C programs). Understanding that the same system can be represented at different levels of abstraction — from gates to C code — was an important insight.

**Suggestions for improvement:** Step-by-step guidance on translating a circuit diagram into C code would help. The conceptual jump between the two representations can be difficult without explicit scaffolding.

**Other thoughts:** Lab 3 was the most demanding of the three labs, but also the most rewarding. Completing both the diagram and the working programs gave a genuine sense of having understood the material at multiple levels.
