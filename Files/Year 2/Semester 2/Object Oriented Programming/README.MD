# 📘 Object Oriented Programming

**Year 2 · Semester 2**

This course formalised object-oriented programming in Java — covering classes, inheritance, polymorphism, encapsulation, interfaces, exception handling, and applied software design through a mini project.

## 📖 What I Learned

Object Oriented Programming in Java consolidated the OOP concepts introduced in C++ and extended them with Java-specific features — interfaces, abstract classes, checked exceptions, and the Java Collections Framework. I learned to design class hierarchies, apply design principles such as single responsibility and open/closed, and structure a larger application across multiple classes.

The mini project required building a complete application that demonstrated OOP principles in a realistic context.

---

## 💭 Reflection

**What I gained:** Fluency in Java and a more mature understanding of object-oriented design. Learning to think in terms of objects, responsibilities, and interfaces — rather than just functions and data — changed how I approach software design problems.

**Suggestions for improvement:** Including design pattern study as part of the course — factory, strategy, observer — would give students more reusable solutions to common design problems and improve code quality on the mini project.

**Other thoughts:** Java's strictness — with explicit types, checked exceptions, and enforced encapsulation — was initially frustrating but ultimately beneficial. It forced me to think more carefully about types, error handling, and access control than Python had. The discipline carried over into better practices in all languages I use since.
