# 📐 Object Oriented Programming — Mini Project

**Year 2 · Semester 2 · Object Oriented Programming**

A Java application developed as the mini project for Object Oriented Programming, demonstrating core OOP design principles in a complete working system.

---

## 🎯 Overview

The mini project required designing and implementing a Java application that applied the OOP concepts covered in the course — class hierarchies, inheritance, polymorphism, encapsulation, and interfaces — to a meaningful problem domain.

---

## 🛠️ Tools and Technologies

- Java programming language
- Object-oriented design principles

---

## 📁 Folder Contents

No source files retained in this folder (submitted externally).

---

## 📖 What I Did

I designed a class structure for the application, implemented the core logic using Java classes and interfaces, handled exceptions appropriately, and tested the system against the project requirements. The project was planned before implementation, with a class diagram produced to guide the design.

---

## 💭 Reflection

**What I gained:** Experience designing and building a complete OOP application from scratch. The project required making real design decisions — which classes to create, what responsibilities they should have, how they should interact — rather than following a prescribed structure.

**Suggestions for improvement:** A design review milestone partway through the project would help catch structural problems before too much code has been written. Changing a class hierarchy late in development is costly and teaches the importance of upfront design.

**Other thoughts:** The mini project was the most independent software development task I had completed to that point. Managing the full process — design, implementation, testing, and documentation — was a meaningful step towards professional-level project work.
