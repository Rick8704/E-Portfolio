# 📘 Data Mining

**Year 2 · Semester 2**

This course explored techniques for discovering patterns, relationships, and insights in large datasets — covering classification, clustering, association rule mining, and model evaluation.

---

## 📂 Work

No assignment or lab subfolders recorded for this course.

---

## 📖 What I Learned

Data Mining introduced the algorithmic methods that turn raw data into actionable knowledge. I learned the core mining tasks — classification (predicting a category), regression (predicting a value), clustering (grouping similar records), and association rule mining (finding co-occurrence patterns). Model evaluation techniques such as confusion matrices, precision/recall, and cross-validation gave me tools for assessing how well a model performs.

The course also covered the CRISP-DM process — a structured framework for data mining projects that has remained useful in subsequent analytical work.

---

## 💭 Reflection

**What I gained:** An understanding of how machine learning and data mining algorithms work at a conceptual level, and how to evaluate their results critically. Knowing that a high accuracy score can be misleading — due to class imbalance or overfitting — is one of the most practically important lessons from this course.

**Suggestions for improvement:** More hands-on implementation using Python (scikit-learn) alongside the theoretical content would have made the concepts more concrete. Understanding an algorithm by implementing it, even simply, is more durable than understanding it by description alone.

**Other thoughts:** Data Mining bridged the gap between data analytics and machine learning for me. Understanding that mining is about finding patterns — not just predicting outcomes — gave me a broader view of what can be done with data beyond supervised learning.
