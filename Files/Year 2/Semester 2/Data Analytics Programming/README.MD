## 📖 What I Learned

Data Analytics Programming was where Python became my primary working language. I learned to load and clean datasets with pandas, explore data using descriptive statistics, produce visualisations with matplotlib and seaborn, and structure an analytical workflow in Jupyter Notebooks.

Across the three assignments, the complexity increased progressively — from basic data loading and exploration to multi-step analysis workflows involving cleaning, transformation, aggregation, and visual communication of results.

---

## 💭 Reflection

**What I gained:** Python proficiency and confidence in the full analytics workflow — from raw data to insight. Working with real datasets in Jupyter Notebooks gave me a practical understanding of how data analysis actually proceeds: iteratively, with constant checking and revision.

**Suggestions for improvement:** Including a module on data quality assessment — dealing with missing values, outliers, and inconsistent formats systematically — would prepare students better for real-world data, which is rarely clean.

**Other thoughts:** Data Analytics Programming was a turning point. Before this course, data was something I read about; after it, data was something I worked with. The hands-on experience of wrangling a messy dataset, discovering patterns in the data, and communicating findings through charts shifted my understanding from theoretical to practical.
