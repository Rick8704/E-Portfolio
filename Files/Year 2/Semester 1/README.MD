## 📖 What I Learned

Data Structures & Algorithms provided essential tools for organising and processing data efficiently — covering arrays, linked lists, trees, graphs, sorting, and searching. This course changed how I approach problem-solving in code.

Database (WBL) was the first deep engagement with relational databases — entity-relationship modelling, SQL, normalisation, and query optimisation. The WBL component connected classroom theory to realistic database design scenarios.

Network Communications introduced the layered model of how data travels across networks, covering protocols, addressing, routing, and security basics. Software Engineering (WBL) covered structured approaches to developing software — requirements, design patterns, version control, and testing practices applied to a team project.

---

## 💭 Reflection

This was the semester where the connection between data and engineering became clear. Learning SQL alongside data structures gave me two complementary ways of thinking about information — how to store it and how to process it efficiently.

The WBL elements in Database and Software Engineering pushed me to work collaboratively, communicate designs clearly, and meet deadlines under realistic project conditions. These experiences were more demanding than pure coursework but also more rewarding.

By the end of this semester, I had a much stronger sense of the tools and practices that define professional data engineering work.
