# 📘 Software Engineering (WBL)

**Year 2 · Semester 1**

This work-based learning course introduced structured software development practices — covering requirements engineering, software design, version control, testing methodologies, and team-based project delivery — applied to the KADA cooperative financial system.

## 📖 What I Learned

Software Engineering (WBL) provided a framework for developing software professionally rather than informally. I learned to gather and document requirements using SRS templates, produce design artefacts including UML diagrams and architecture documents, apply version control with Git throughout the project, write test cases against defined system behaviours, and manage a software project through its full lifecycle.

The WBL component required applying these practices to the KADA system — a real cooperative management application — which meant that design decisions had genuine consequences and documentation had to be comprehensive enough to guide actual implementation.

---

## 💭 Reflection

**What I gained:** A disciplined approach to software development grounded in real project experience. Writing proper SRS and SDD documents for a working system — rather than as abstract exercises — showed me what good documentation looks like and why it matters for team coordination and long-term maintainability.

**Suggestions for improvement:** More exposure to Agile practices — sprint planning, daily standups, and retrospectives — would better reflect current industry reality. The course introduced Agile concepts, but deeper practice with iterative delivery cycles would strengthen preparation for professional team environments.

**Other thoughts:** Software Engineering taught me that how you develop software matters as much as what you build. A project with clear requirements, documented design decisions, and a version-controlled codebase is far easier to maintain, extend, and hand over than one built informally. The KADA project demonstrated this concretely — the documentation we produced made onboarding new contributors and revisiting earlier decisions significantly smoother.
