# 📘 Software Engineering (WBL)

**Year 2 · Semester 1**

This work-based learning course introduced structured software development practices — covering requirements engineering, software design, version control, testing methodologies, and team-based project delivery.

---

## 📂 Work

| Project | Repository |
|---|---|
| KADA Financial System | [github.com/lauyankai/KADA-system](https://github.com/lauyankai/KADA-system) |

A cooperative financial management system built for Koperasi KADA Kelantan — developed as the WBL team project. The system manages member records, financial transactions, loan tracking, and reporting through a structured web application built collaboratively across the semester.

---

## 📖 What I Learned

Software Engineering (WBL) provided a framework for developing software professionally rather than informally. I learned to gather and document requirements, produce design artefacts (UML diagrams, architecture documents), apply version control with Git, write test cases, and manage a software project through its lifecycle.

The WBL component required applying these practices to a real team project — providing experience of the coordination, communication, and iteration that professional software development involves.

---

## 💭 Reflection

**What I gained:** A disciplined approach to software development. Understanding the full process — from requirements to delivery — and the role of documentation, testing, and version control in making that process reliable has shaped how I approach every project.

**Suggestions for improvement:** More exposure to Agile practices — sprint planning, daily standups, retrospectives — would better reflect current industry practice. The structured waterfall-oriented content is useful, but most professional teams work iteratively.

**Other thoughts:** Software Engineering taught me that the way you develop software matters as much as the software itself. A well-structured project with clear requirements, version-controlled code, and a test suite is far easier to maintain, extend, and hand over than one that was built quickly without these practices. That lesson has been reinforced repeatedly in subsequent project work.
