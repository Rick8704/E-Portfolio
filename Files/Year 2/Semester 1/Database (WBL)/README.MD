# 📘 Database (WBL)

**Year 2 · Semester 1**

This work-based learning course introduced relational database design — covering entity-relationship modelling, SQL, normalisation, and query optimisation applied to realistic database scenarios.

---

## 📂 Work

No assignment or lab subfolders recorded for this course.

---

## 📖 What I Learned

Database (WBL) provided the foundation for all subsequent data engineering work. I learned to design relational schemas using entity-relationship diagrams, normalise tables to eliminate redundancy, write SQL queries ranging from basic SELECT statements to complex multi-table joins, and think about indexing and query performance.

The WBL component connected these skills to realistic database design scenarios, requiring me to make design decisions and justify them against real-world requirements.

---

## 💭 Reflection

**What I gained:** A solid foundation in relational database design and SQL that I have used in every data project since. Understanding normalisation — and when to deliberately denormalise for performance — is a skill that remains directly applicable.

**Suggestions for improvement:** Including a module on database performance tuning and query execution plans earlier would prepare students better for the challenges they encounter when working with large datasets in later courses.

**Other thoughts:** Database is the course that most directly enabled the rest of my data engineering studies. Without a strong understanding of how relational data is structured and queried, courses on data analytics, data mining, and data pipelines would have been much harder to follow. The investment in this foundation has paid off consistently.
