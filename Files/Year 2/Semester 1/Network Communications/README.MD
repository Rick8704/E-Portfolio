## 📖 What I Learned

Network Communications explained how devices communicate across local and wide area networks. I learned the role of each OSI layer, how IP addressing and subnetting work, how routers and switches forward traffic, and how protocols like TCP, UDP, HTTP, and DNS operate. Packet Tracer labs provided hands-on experience building and troubleshooting simulated networks.

---

## 💭 Reflection

**What I gained:** A working understanding of how data moves between systems — knowledge that supports cloud architecture, distributed data pipelines, and any work involving networked services. Understanding why a connection fails or why latency spikes requires the mental model that this course built.

**Suggestions for improvement:** More real-device lab sessions — even with basic routers and switches — would complement the Packet Tracer simulations. Physical equipment introduces practical considerations that simulators sometimes hide.

**Other thoughts:** Network Communications is often undervalued in data engineering curricula, but it directly supports cloud and distributed systems work. Understanding how Azure, AWS, and Spark nodes communicate requires the network foundation that this course provided.
