## 📖 What I Learned

Data Structures & Algorithms transformed how I think about organising and processing information in code. I learned when to use different structures — arrays for indexed access, linked lists for flexible insertion, trees for hierarchical data, graphs for networks — and which algorithms operate efficiently on each.

Complexity analysis (Big-O notation) gave me a framework for evaluating the efficiency of a solution beyond just whether it produces the correct output.

---

## 💭 Reflection

**What I gained:** The ability to reason about trade-offs between data structures and algorithms. Choosing the right structure for a problem — and understanding why — is now a habit that I apply in data engineering, where the choice of storage format or processing approach can have significant performance consequences.

**Suggestions for improvement:** More visualisation tools for tree and graph structures during lectures would help. Seeing how a tree rotates during insertion, or how BFS and DFS explore a graph, is far clearer visually than in code alone.

**Other thoughts:** The flight simulation assignment was the most engaging task in this course. Applying data structures to a realistic simulation scenario — tracking flights, routes, and scheduling — made the abstract concepts feel relevant to real engineering problems.
