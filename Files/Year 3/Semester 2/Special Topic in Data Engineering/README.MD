## 📖 What I Learned

Special Topic in Data Engineering brought together the most advanced data engineering content of the degree. Azure tutorials introduced cloud data storage — blob storage, data lake, and synapse analytics. The Apache Spark tutorial covered distributed data transformation. The AI Algorithm tutorial examined how neural network models are evaluated and compared.

The Azure Tutorial and DuckDB/dbt work provided experience with the analytical pipeline — from raw data ingestion through transformation, validation, and output. The PPG industry project applied these concepts in a real-world context, requiring the integration of multiple tools and techniques to address an actual industry problem.

---

## 💭 Reflection

**What I gained:** A working knowledge of the modern data engineering stack — cloud storage, distributed processing, analytical databases, transformation frameworks, and AI-driven pipelines. This course was the closest to professional data engineering practice of any course in the degree.

**Suggestions for improvement:** More time to work with each tool before moving to the next would improve depth of understanding. The breadth of the course is valuable, but some tools received limited hands-on time. Extended project phases or optional deep-dive tasks would help students who want to go further with specific technologies.

**Other thoughts:** Special Topic felt like the point where everything came together. The tools, concepts, and practices from three years of study all appeared in some form in this course — applied together to real data engineering problems. It showed me what data engineering looks like in practice and confirmed that this is the field I want to work in.
