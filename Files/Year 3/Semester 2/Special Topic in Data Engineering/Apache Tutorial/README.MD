## 💭 Reflection

**What I gained:** Practical experience with Apache Spark — one of the most widely used tools in production data engineering. Understanding how Spark distributes work across a cluster, how lazy evaluation affects when computations happen, and how to structure a PySpark workflow is directly applicable to large-scale data pipeline work.

**Suggestions for improvement:** Running the tutorial on an actual distributed cluster — even a small cloud-based one — rather than local mode would provide a more realistic experience and expose the network and coordination costs that local Spark does not simulate.

**Other thoughts:** The Spark tutorial was one of the most valuable technical experiences of the degree. Moving from single-machine pandas to distributed Spark processing is a conceptual shift as well as a technical one — the tutorial made that shift concrete and manageable.
