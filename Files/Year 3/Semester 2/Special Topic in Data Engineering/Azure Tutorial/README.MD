I worked through the Azure tutorial workflow — provisioning cloud storage resources, uploading and organising data in Blob Storage and/or Data Lake, and performing basic operations through the Azure portal or SDK. ## 💭 Reflection

**What I gained:** Hands-on experience with cloud data storage — one of the most fundamental components of modern data engineering infrastructure. Understanding the difference between object storage, data lakes, and analytical databases, and knowing when to use each, is directly applicable to cloud data architecture decisions.

**Suggestions for improvement:** Extending the tutorial to include a simple data pipeline — ingesting data into Blob Storage, transforming it, and loading it to a downstream service — would demonstrate how Azure storage integrates into a complete workflow rather than existing in isolation.

**Other thoughts:** The Azure tutorial was a turning point in understanding cloud data engineering. Moving from local file storage to cloud object storage requires a shift in how you think about data organisation, access, and cost. The experience of actually provisioning and using cloud resources made those concepts concrete in a way that reading about them does not.
