# 📘 Application Development

**Year 3 · Semester 2**

This course covered mobile application development using Flutter and Dart — including local storage, Firebase Authentication, Firestore, role-based access, and application testing.

---

## 📂 Work

No assignment or lab subfolders recorded for this course.

---

## 📖 What I Learned

Application Development introduced the full mobile development workflow using Flutter. I learned to build cross-platform mobile applications in Dart, structure UI components using Flutter widgets, persist data locally, integrate Firebase for authentication and cloud storage, implement role-based access for different user types, and apply testing practices to verify application behaviour.

The course provided practical experience of the complete application development cycle — from design to deployment-ready build.

---

## 💭 Reflection

**What I gained:** The ability to build a functional mobile application end-to-end. Understanding how Firebase integrates with a Flutter app — handling authentication, real-time data, and role-based access — gave me a complete picture of a modern mobile backend architecture.

**Suggestions for improvement:** More time spent on application testing and debugging techniques would be valuable. Testing a UI-heavy application requires different approaches from testing backend logic, and systematic test-writing for mobile apps is a skill that needs dedicated practice.

**Other thoughts:** Application Development showed me that good mobile software is about more than writing code that runs — it is about designing an experience that is fast, reliable, and intuitive on a small screen with intermittent connectivity. Those constraints require careful thinking about state management, data loading, and error handling that pushed me to write more defensive and considerate code.
