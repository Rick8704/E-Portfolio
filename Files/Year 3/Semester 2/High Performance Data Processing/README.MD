# 📘 High Performance Data Processing

**Year 3 · Semester 2**

This course introduced techniques for processing large datasets efficiently — covering memory-aware Python processing, performance benchmarking, asynchronous web crawling, and PySpark data preparation.

## 📂 Work

No assignment or lab subfolders recorded for this course.

---

## 📖 What I Learned

High Performance Data Processing addressed the scalability challenges that arise when working with large volumes of data. I learned to write memory-efficient Python code, measure and compare the performance of different processing approaches, implement asynchronous web crawling to collect data without blocking, and use PySpark to process datasets in a distributed computing context.

The course provided practical tools for moving from single-machine data work to distributed processing — a critical transition in data engineering.

---

## 💭 Reflection

**What I gained:** Awareness of performance as a design consideration in data processing. Understanding that the approach chosen — iterative processing, chunked reading, parallel execution, or distributed computing — has measurable consequences for time and resource usage changed how I think about data pipeline design.

**Suggestions for improvement:** More hands-on time with a real Spark cluster — rather than local Spark simulation — would better prepare students for the realities of distributed processing. Local Spark hides many of the network and coordination costs that matter in production.

**Other thoughts:** This course reframed data engineering for me. It showed that getting correct output is only the first requirement — getting it efficiently, at scale, and within resource constraints is the actual engineering challenge. That perspective has shaped how I approach every data processing task since.
