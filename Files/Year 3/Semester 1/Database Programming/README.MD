## 📖 What I Learned

Database Programming went significantly beyond the SQL covered in Year 2. I learned to write stored procedures, triggers, and functions that embed application logic inside the database; use views to simplify complex queries; manage transactions to ensure data consistency; and analyse query execution plans to identify and resolve performance problems.

The two projects required applying these techniques to realistic database scenarios, producing schemas, queries, and procedures that could support real applications.

---

## 💭 Reflection

**What I gained:** Advanced SQL skills and a deeper understanding of how databases support complex application logic. Knowing how to use stored procedures, transactions, and indexes — and when each is appropriate — has been directly useful in data engineering pipeline work.

**Suggestions for improvement:** Introducing NoSQL databases as a contrast to relational systems would provide valuable perspective. Understanding when to use a document store or a column store instead of a relational database is an important decision in data engineering, and exposure to that trade-off earlier would be helpful.

**Other thoughts:** Database Programming changed my relationship with SQL. Previously I saw it as a query language; after this course I saw it as a full programming environment with its own logic, error handling, and performance characteristics. That shift made me more effective at designing data systems.
