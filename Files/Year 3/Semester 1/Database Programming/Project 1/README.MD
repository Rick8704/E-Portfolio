## 🎯 Overview

Project 1 required designing a relational database schema for a given domain, implementing the schema, populating it with data, and writing SQL queries and procedures to support defined application requirements.

## 📖 What I Did

I designed the entity-relationship model for the project domain, translated it into a normalised relational schema, wrote the DDL to create the tables, populated the database with sample data, and implemented SQL queries and stored procedures to meet the project requirements. The full process was documented in the project report.

---

## 💭 Reflection

**What I gained:** Experience completing a full database design and implementation project from requirements to working system. Making real design decisions — about normalisation, key choices, and index placement — and then implementing them gave practical insight that reading about database design cannot provide.

**Suggestions for improvement:** A requirement to load and query a realistically sized dataset — with thousands of rows rather than sample data — would introduce the performance considerations that matter in real applications.

**Other thoughts:** Project 1 was the point where database design stopped being abstract. Designing a schema that actually has to support queries efficiently, handle data integrity constraints, and be comprehensible to someone else forced me to think more carefully than any homework exercise had.
